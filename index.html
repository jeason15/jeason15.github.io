MTH 366 Neural Network Lab Python Conversion:¶

Environment Setup:¶

- Download and install miniconda (if you already have anaconda, skip this step)
    - Check for existing conda installation by going into the terminal or command prompt and typing $conda --version
        - If you have Conda, it will tell you the version you have. If you dont, it will give you an error.
    - Download Link: https://docs.conda.io/en/latest/miniconda.html
    - Select your operating system from the "Latest Miniconda Installer Links" section. Follow the on-screen prompts
    - If miniconda installer asks to add conda to path, select yes. 

- Once you have conda:
    - Open a terminal and type the following commands (everything after the "$"):
        - $conda create -n <your env name here>
            - substitute what you want to call your environment in the <your enve name here> above, for instance, I used 366NueralNetLab
        - $conda activate <your env name here>
        - $conda install python=3.7
        - $conda install jupyter
        - $conda install keras
        - $conda install tensorflow=2.0
        - $conda install matplotlib
        - $conda install scikit-learn
    - Close your terminal window

Creating and Running a Python Notebook:¶

- Open a new terminal window, type the following commands:
    - $conda activate <your env name here>
    - $cd <filepath you will want to use when you save and reload your work>
        - Example -> $cd ~/Documents/MTH366/Labs 
    - $jupyter notebook

- This will launch a new jupyter server and launch the front end client in your web browser.
- In the browser, select the "new" box to the right of the screen
- Select the option under the "notebook" section that says "Python3 (ipykernel)"
    - Note: Because you launched jupyter from the terminal where you had your conda environment with the packages you needed activated, 
            this allows the default iPykernel to find the correct version of python and all of the packages that you will need to run the code.
- When your new jupyter notebook launches, you can (and should): file -> save as 
- Save your notebook as the .ipynb filepath. This will allow it to run later if you need to work on it.

Running a Previously Created Jupyter Notebook:¶

- Open a terminal and type the following:
    - $conda activate <your env name here>
    - $cd <filepath you specified above when you created the notebook>
        - Example -> $cd ~/Documents/MTH366/Labs 
    - $jupyter notebook
- Like before, the jupyter server will start, and the front end client will launch in your default web browser.
- Find your previously saved Jupyter notebook (the ".ipynb" file from above) and click on it to open it

Developing in Jupyter:¶

- Jupyter works very much like R-Markdown, you can create code in chunks, create markdown chunks for plain text, etc. The overall feel of your jupyter will be very similar to what you are already used to. If you are having trouble, there are tons of tutorials on jupyter.

# For training a model, there are a few popular ways to run all the samples through. First, a definition...
# epoch = a full forward and backward pass through the model of each of the training samples
# iterations = how many times you need to train the model to complete an epoch
# So, if your batch size is 1, then you will need n=num samples (60000) iterations for this dataset in order to complete a single epoch
# Now, we will define the three methods...
# Batch: large batches of the training data are processed at once. This allows for better convergence of the minimum of the loss function, but is memory intensive, 
# as each of the iterations must store more of the data in memory
# Mini-batch: smaller batches are used to allow training data to fit into memory for each iteration, not as good as batch, but sometimes necessary.
# Stochastic: mini-batch with only a single sample.

GPU Acceleration¶

If you have a discreet Graphical Processor Unit (GPU) on your machine, then you can make use of that to speed up the training of your models by a pretty considerable amount. There are some caveats, though...

  • Type of GPU: Currently, the most popular libraries (Tensorflow and Torch) used for neural networks support onlu GPU's build using something called CUDA. This is an Nvidia product. This means that you will only have access to the GPU if it is an Nvidia GPU (Examples: GTX, RTX, etc.).
  • Compatibility Problems: Because your instructions are offloaded to a specific type of processor, the drivers for that device, as well as some libraries used to support that device are crucial. What does this mean for you? Well, it is highly likely that whatever device drivers are installed on your machine at this time will have compatibility problems.

So, what is the point of this??? The point here is that if you are training models that are very large, this is an option. My best advice is to do your homework. Utilize virtual environments as much as possible, and consult the library documentation for the compatibility list.
